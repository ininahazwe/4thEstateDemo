# Development Preferences

## Language & Localization

### UI/Content Language
**Preference**: English (en)
- All metadata descriptions and labels in English
- All JSON-LD schema descriptions in English
- All comments in code files in English
- User-facing content uses en_GH locale

### Default Locale for Metadata
- `locale: "en_GH"` for Open Graph
- `og:locale: "en_GH"` for social media

### Language Alternates
- Primary: English (en) - Default
- Secondary: French (fr) - Available at `/fr` path

## Code Standards

### Comments
- All code comments: English
- All documentation: English
- All error messages in user-facing code: English

### File Naming
- Use English for file names
- Use snake_case for multi-word names

### Variable/Function Naming
- Use English names for all variables and functions
- Use camelCase for JavaScript/TypeScript

## Future Requests

For all future development work:
- Assume English as the default language unless specified otherwise
- Apply these language preferences to all new metadata, descriptions, and documentation
- When creating new pages/components, follow these English-first conventions
